# ClothApp_Manual_Testing
Manual testing of a clothing e-commerce website covering test case creation, execution, and defect reporting to ensure smooth user experience.

1. Project Title
Cloth Shopping Web Application – Manual Testing 
2. Project Overview
This project involves the manual testing of an Online Cloth Shopping Application. The goal is to validate the end-to-end functionality to ensure that all features are functioning correctly and users have a smooth experience while registering, browsing, and purchasing clothing items.
3. Features Tested
•	User Registration 
•	Login Module 
•	Product Browsing & Wishlist 
•	Shopping Cart 
•	Checkout & Payment 
•	Contact Us & Account Deletion 
4. Testing Type
•	Functional Testing 
•	Validation Testing 
•	Usability Testing 
•	System Testing 
   
5. Tools Used
•	Microsoft Excel – For writing manual test cases, RTM, test data, and tracking execution status.
•	Microsoft Word – For preparing and documenting the Test Plan and Test Summary Report.
•	Google Chrome – For manually testing the web application across different scenarios.
6. Test Artifacts
•	Test Plan  
•	RTM Document 
•	Test Cases 
•	Test Data 
•	Defect Report 
•	Test Summary Report 
7. Test Execution Summary
•	Total Test Cases: 60 
•	Executed: 60 
•	Passed: 47 
•	Failed: 13 
•	Defects Raised: 13 
8. Defects Found
•	Registration: DOB field allows submission without Year selected.
•	Cart: Quantity update not reflected in total; system accepts 0 or negative quantity.
•	Checkout: Invalid card numbers accepted; special characters allowed in payment name fields.
•	Account Delete: Account gets deleted instantly without any confirmation prompt.
•	Product: Sorting options not functioning; Back button issues.
9. Learning Outcomes
•	Practical understanding of Manual Testing methodologies.
•	Experience in designing and executing Positive and Negative test cases.
•	Expertise in Logging defects with severity and priority details.
•	Performing end-to-end System Testing on a live e-commerce environment.
